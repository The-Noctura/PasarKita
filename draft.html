Dokumen Perencanaan Project

PasarKita

Sistem Pemesanan Sayur & Ayam — Notif Telegram
2 Orang · Frontend & Backend PHP Native · MySQL 7 Minggu SMKN 1 Katapang
01

Gambaran Sistem

Website ini bukan toko online penuh. Fungsinya sebagai jembatan — pelanggan buat akun, lihat produk, lalu isi form checkout. Hasilnya tersimpan ke database dan notifikasi otomatis dikirim ke Telegram admin. Konfirmasi, negosiasi, dan deal tetap dilakukan manual antara admin dan pelanggan. Admin kelola semua data pesanan lewat panel.

✓ Sederhana, realistis, dan langsung menyelesaikan masalah utama klien — pesanan tercatat rapi dan nganjuk bisa dilacak.
02

Alur Sistem

Sisi Pelanggan

1
Buka website, lihat produk
Bisa tanpa login. Halaman beranda dan produk terbuka untuk semua.
2
Daftar atau login
Diwajibkan sebelum bisa checkout. Data nama dan HP otomatis terisi dari akun.
3
Isi form checkout
Deskripsi pesanan bebas (textarea), alamat pengiriman, metode bayar (transfer / tunai).
4
Submit → pesanan tersimpan + notif Telegram terkirim
Data masuk ke tabel transaksi. Admin langsung dapat pesan di Telegram.
5
Halaman konfirmasi muncul
Pelanggan lihat ringkasan pesanan dan info bahwa admin akan segera menghubungi.

Sisi Admin

1
Terima notif di Telegram
Berisi nama, HP, deskripsi pesanan, alamat, dan metode bayar.
2
Hubungi pelanggan, deal harga
Dilakukan di luar sistem — WA atau telepon langsung.
3
Buka panel admin, input total harga
Total harga diisi manual setelah deal selesai.
4
Tandai status: lunas atau hutang → selesai
Kalau hutang, otomatis masuk ke halaman daftar hutang.
Contoh isi notif Telegram yang diterima admin
📦 Pesanan Baru Masuk! Nama : Ibu Sari HP : 08123456789 Alamat : Jl. Kenanga No.5, RT 02 Pesanan : 1 ekor ayam, 2 ikat kangkung, 1 kg wortel Bayar : Transfer Buka panel → pasarkita.com/admin/pesanan.php
03

Halaman yang Dibuat

Publik — Sisi Pelanggan
index.php
Beranda
Hero/banner, ringkasan produk unggulan, info singkat cara pesan.
produk.php
Daftar Produk
Produk dari database — nama, harga, satuan, foto. Tombol "Pesan Sekarang" mengarah ke checkout.
contact.php
Kontak
Nomor WA, jam operasional, area pengiriman. Bisa tambah embed Google Maps lokasi pasar.
checkout.php
Form Checkout
Deskripsi pesanan (textarea bebas), alamat antar, metode bayar. Nama & HP otomatis dari sesi login. Submit → simpan DB → kirim notif Telegram → halaman konfirmasi.
daftar.php
Registrasi
Form daftar akun baru: nama, email, password, nomor HP, alamat.
login.php
Login Pelanggan
Terpisah dari login admin. Cek role setelah berhasil masuk.
Admin — Sisi Pemilik Toko
admin/login.php
Login Admin
Cek role = admin. Redirect ke dashboard jika berhasil.
admin/pesanan.php
Daftar Pesanan
Semua pesanan masuk — nama, HP, deskripsi, status, tanggal. Klik untuk input total harga dan ubah status.
admin/produk.php
Kelola Produk
CRUD produk. Satu form untuk tambah dan edit. Kolom aktif untuk hide produk tanpa hapus data.
admin/laporan.php
Laporan
Total pesanan masuk dan total pemasukan (SUM total_harga). Filter harian/mingguan/bulanan.
admin/hutang.php
Daftar Hutang
Filter pesanan dengan status_bayar = hutang. Tampil nama, HP, total. Tombol "Tandai Lunas".
ℹ Total 11 file utama + logout.php + koneksi.php. Jumlah yang realistis untuk 2 orang dalam 7 minggu.
04

Database

3 tabel, cukup untuk semua fitur yang direncanakan.

users — login pelanggan & admin, satu tabel
id nama email password hp alamat role created_at
transaksi — inti sistem
id user_id deskripsi_pembelian alamat_antar metode_bayar total_harga (nullable) status status_bayar tgl_pesan updated_at
produk — katalog yang dikelola admin
id nama kategori harga satuan foto aktif created_at
pk Primary key fk Foreign key penting Kolom kritis nullable Boleh kosong
⚠ total_harga dibiarkan NULL saat pelanggan submit — diisi admin setelah deal di WA. Tanpa ini laporan pemasukan tidak bisa dihitung.
05

Integrasi Telegram

Tidak butuh library eksternal. PHP punya curl bawaan yang cukup untuk kirim pesan ke Telegram Bot API.

Yang perlu disiapkan di awal:

1
Buat bot via @BotFather di Telegram
Dapat BOT_TOKEN — simpan di file konfigurasi, jangan di-commit ke GitHub.
2
Dapatkan CHAT_ID akun Telegram admin
Bisa pakai @userinfobot untuk cari tahu CHAT_ID.
3
Simpan keduanya di config.php
Tambahkan config.php ke .gitignore agar tidak terupload ke repo publik.
06

Pembagian Tugas

Dibagi berdasarkan layer, bukan fitur — supaya tidak tabrakan di file yang sama.

FE
Anggota 1 — Frontend
Semua tampilan yang dilihat pelanggan
  • Setup Bootstrap, header.php, footer.php, style.css
  • index.php — beranda (hero, produk unggulan)
  • produk.php — grid produk dari DB (tampilan saja)
  • contact.php — info kontak & area pengiriman
  • checkout.php — tampilan form (form HTML saja)
  • daftar.php & login.php — tampilan form (HTML saja)
  • Semua halaman admin — layout tabel & form (Bootstrap)
  • Pastikan semua halaman tampil wajar di HP
BE
Anggota 2 — Backend
Semua logika PHP dan database
  • Setup XAMPP, buat database & semua tabel, koneksi.php
  • Setup GitHub repo, buat struktur folder
  • Logika registrasi & login (session, password_hash)
  • Logika checkout — simpan ke DB + kirim notif Telegram
  • Query produk untuk ditampilkan di produk.php
  • Semua logika admin — CRUD pesanan, produk, laporan, hutang
  • Proteksi halaman admin (cek session & role)
⚠ Frontend jangan hardcode data — pakai variabel PHP yang disediakan backend. Backend jangan pegang tampilan — serahkan ke file HTML/CSS frontend. Komunikasi lewat variabel dan include.
